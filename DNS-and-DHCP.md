# DNS and DHCP

## Objective

To understand the purpose of DNS (Domain Name System) and DHCP (Dynamic Host Configuration Protocol) in computer networks.

## What is DNS?

DNS (Domain Name System) translates domain names into IP addresses so that computers can communicate over a network.

Example:

* Domain Name: [www.google.com](http://www.google.com)
* IP Address: 142.250.183.78

Without DNS, users would have to remember IP addresses instead of website names.

## Functions of DNS

* Resolves domain names to IP addresses
* Simplifies Internet browsing
* Supports website accessibility
* Reduces the need to memorize IP addresses

## What is DHCP?

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses and other network settings to devices connected to a network.

## Information Provided by DHCP

* IP Address
* Subnet Mask
* Default Gateway
* DNS Server Address

## Advantages of DHCP

* Automatic IP address assignment
* Reduces manual configuration
* Prevents IP address conflicts
* Simplifies network administration

## DNS vs DHCP

| DNS                                   | DHCP                               |
| ------------------------------------- | ---------------------------------- |
| Resolves domain names to IP addresses | Assigns IP addresses automatically |
| Helps locate websites and servers     | Configures devices on the network  |
| Uses domain names                     | Uses IP configuration              |

## Why DNS and DHCP are Important

* Enable efficient network communication.
* Simplify network management.
* Reduce configuration errors.
* Essential services in enterprise networks.
